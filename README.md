# Ответы на открытые вопросы
## "Нативные" технологии
### HTML
❓ 1. Форма доступна для людей с ограниченными возможностями, для этого реализованы следующие функции:
- подсветка (обводка синим цветом) активного элемента - помогает пользователю понять, где он находится;
- автоматическая навигация по tab (семантические html-элементы) - пользователь может пройтись по кнопкам ввода без мыши, используя tab;

❓ 2. Обоснование подхода к верстке: форма оформлена с использованием flex, так как необходимо выравнивание по центру, элементы выстроены вертикально, с помощью данного подхода удобно расположить элементы таким образом и обеспечить адаптивность.

### CSS
Добавлена  transition-анимацию на поля ввода (подсветка некорректного ввода), добавлена keyframe-анимация на поля ввода ("дрожащий" инпут на неверный ввод), добавлены конфетти на фон.

❓ 1. Media queries для адаптивной верстки:
Были выбраны типичные диапазоны для различных размеров экранов:
Для мобильных устройств - 0 - 480px
Для планшетов стандартный брейкпоинт - 481 - 768px
От 769px - компьютеры, большие планшеты и т. д.

### JS
❓ 1. Для сохранения данных используется localStorage: данные остаются и после закрытия страницы, поэтому при перезагрузке или при повтором просмотре появляется уведомление с отображением логина пользователя.

❓ 2. Использовала как стрелочные, так и именованные функции, больше понравились стрелочные, так как они меньше по объему и удобны для коротких действий.

## Инфраструктура

❓ 1. Использовала менеджер пакетов npm, так как он является универсальным, имеет интеграцию с Node и подходит для простых проектов.

❓ 2. Выбрала готовую конфигурацию с использованием команды npx tsc --init. Готовые настройки являются подходящими и универсальными для большинства проектов, поэтому для максимальной строгости проверки типов были добавлены параметры:

"noImplicitAny": true, - для улучшения типизации
"strictNullChecks": true, - контроль null-значений

Типы данных описаны в файле type.ts

❓ 3. Выбор сборщика - Vite
Использовала Vite, так как он простой в настройке, имеет встроенную поддержку TS и высокую скорость работы.

❓ 4. Выбрала готовую конфигурацию с использованием команды npx eslint --init, так как готовая конфигурация предлагает оптимальные настройки и соответствует большему количеству требований.

❓ 5. Для настройки конфигурации использовала команду npm init prettier, через терминал были выбраны необходимые стандартные настройки, которые необходимы для форматирования кода.

❓ 6. Для тестов выбрала Vitest и написала простой тест проверки функции. Выбрала готовую конфигурацию, только указала путь к тестам. Выбрала его, потому что легко настраивается и уже используется сборщик Vite - легко интегрируется.
